# Mニュース 開発ルール

## 進行ルール(プロジェクト共通)
- コストがかかるもの(有料API、有料プラン、外部サービス課金等)は勝手に判断せず、必ず事前に確認を取る
- 法的に問題のあることはしない
- セキュリティ的にリスクのある実装(認証なしの管理画面公開、秘密情報のクライアント露出等)は勝手に判断せず確認を取る
- 人間の判断が必要なものはブロックせず後回しにして、進められる部分を先に進める
- 利用量の管理(Vercel等のインフラ + Claude自体の利用枠の両方):
  - 週間/セッションの利用枠を序盤で使い切るペースにしない(例: 初日に週間枠の50%超えはNG)
  - 大きいタスクは一括でやらず、優先度順に区切って実行し、区切りごとに利用ペースを踏まえて続行可否を判断する
  - トークン消費が大きい作業は対象を絞る・サンプリングする
  - 利用量が想定より膨らみそうなら途中で止めて報告する

## 本番サービスの保護(夜間作業でも閲覧者がいる前提で扱う)
- 本番(www.mnews.jp)を落とさないこと
- mainブランチへの直接pushはしない。必ず作業ブランチ + Vercelのプレビューデプロイで動作確認してから本番反映する
- データ変更(選手DB・イベントデータ・結果データのスキーマ変更や一括更新)は実行前にバックアップを取り、切り戻し手順を用意してから行う
- 本番反映は「ビルドが通っている + プレビューで主要ページ(トップ/選手/イベント/結果/rankings)の表示確認済み」の場合のみ
- 本番反映後、主要ページが正常表示されているか確認してから次の作業に進む。エラー(ビルド失敗、500、白画面)が出た場合は直前のデプロイに即ロールバックし、原因調査はその後に行う
- 複数タスクの本番反映をまとめて一度に行わない(タスク単位でデプロイを分ける)
- 通常の本番反映は原則深夜帯(閲覧者が少ない時間帯)に行う。ただし一般ユーザーに影響が出ているデグレの修正は緊急対応として時間帯を問わず即時デプロイしてよい
- cron(daily-digest, JST 8:00 = UTC 23:00)の予定時刻付近に本番デプロイを重ねるとその日のcron実行がスキップされることがある。8:00前後のデプロイは避け、重なった日は翌朝メール到達を確認する

## デプロイ
- 通常: `git push origin main && vercel deploy --prod --yes`
- 上記の本番保護ルールに該当する変更(大きい機能追加・データ変更等)は作業ブランチ + `vercel deploy`(プレビュー)で確認してからmainにマージする

## 認証境界(重要)
- 認証必須は `/admin/*`(管理画面)と `/api/admin/*`(管理系API)のみ
- 一般ユーザーが使う機能は絶対に `/admin` 配下に置かない(middlewareに巻き込まれ一般ユーザーがログイン画面に飛ばされるデグレになる):
  - X投稿用カード作成ツール → `/tools/fighter-card`
  - OG/シェア画像生成API → `/api/og/*`

## コンテンツ取得・著作権ポリシー（必須遵守 / 例外なし）

### 取得ソースのルール
- **eFight（efight.jp）: 取得禁止。**
  規約で「蓄積」「スクレイピング（非正規手法での情報取得）」を明示的に禁止しており、かつ有料課金メディアのため。自動取得ジョブは稼働させない。既存の取り込み済みアイテムは表示側で除外する（取得ジョブ停止＋表示除外で対応）。
  DBからの物理削除はこのポリシーでは行わない（不可逆操作のため人間の明示判断で別途実施）。
- **ORICON（oricon.co.jp）: RIZIN関連のみ取得可。**
  取得元は格闘技タグ一覧（https://www.oricon.co.jp/news/tag/id/kakutougi/）。
  `RIZIN` / `ライジン` のキーワード一致でフィルタし、非RIZIN記事は除外する。
  選手名による追加フィルタは、手書きリストの保守コストを避けるため現時点では実装しない（保留）。
  将来必要になった場合は、FIGHTERS から org==RIZIN を動的に引く形で実装する（手書き選手名リストは作らない）。
  ORICONは規約上「リンクは自由・本文/画像の複製は禁止」→ 見出し＋元記事URLのみ格納。
- **MMAPLANET / ゴング格闘技 / その他商用メディア:**
  見出し＋元記事URLのみ。本文・画像は扱わない。

### 全ソース共通の取得ルール
- 保存してよいのは「見出しテキスト」＋「元記事URL」のみ。**本文・要約・画像は取得も保存もしない。**
- 見出しは元媒体のデッドコピー（丸写し）を避け、mnews側の言葉で書き直す。
- 表示は必ず元記事へのリンク送りにし、流入を元媒体へ返す（代替・競合しない）。
- 【根拠】ニュース見出し自体に著作権は認められにくいが、「鮮度の高い見出しを営利で反復継続してデッドコピーし、元媒体の業務と競合する形で配信する行為」は不法行為と判断された判例がある（読売オンライン見出し事件・知財高裁 平成17年10月6日）。書き直し＋リンク送りでこれを回避する。

### 選手画像ポリシー
- **選手写真の無断転載をしない。** 取得・保存・表示のいずれも不可。
  使用可なのは、主催公式プレスキット等で明示許諾がある画像、またはライセンス素材のみ。
- 選手・対戦カードのビジュアルは自作スタッツカード（画像非依存）で対応する。
  これは法的にクリーンかつ差別化要素になるため、方針として維持する。

## 大会予定（/events）登録ルール

大会予定に「入れる/入れない」は団体ごとに以下で判定する。配信ブランド（Lemino等の冠）ではなく**主催**で判定するのが原則。判断に迷ったらこのルールを正とし、大会ごとに再判断しない。

### 判定ルール（4団体）

- **RIZIN**：全興行を登録。ナンバーシリーズ、LANDMARK、TRIGGER 等すべて含む。
- **修斗**：サステイン主催のプロフェッショナル修斗公式戦（後楽園/ニューピア本興行＝Vol.ナンバー）＋ Lemino修斗のナンバー興行のみ。
  - 主催で判定する。地方主催（THE BLACK BELT JAPAN / Torao Nation State / SECOND OUT 等）は Lemino 冠が付いていても**除外**。
  - 例：`Lemino修斗～POUND OUT～`（THE BLACK BELT JAPAN 主催）は除外。
- **パンクラス**：PANCRASE ナンバー本興行のみ。地方・チャレンジ・アマ系は除外。
- **DEEP**：DEEP本体のナンバー興行（DEEP ○○ IMPACT 等）＋ DEEP TOKYO IMPACT シリーズ ＋ DEEP JEWELS。DEEP Fight Challenge・DEEP☆FUTURE・地方・アマ系は除外。

### 原則

- **主催ベース判定**：配信ブランドや大会名の冠ではなく、主催団体で本興行/地方興行を切り分ける。
- **網羅の対象は大会予定（1行）まで**。結果まとめの記事化は別基準（主要大会＋タイトルマッチ＋DB掲載選手が絡む試合）で選別し、全大会をフル記事化しない。
- データ捏造禁止（会場未確定等は空/未定で扱い、具体名を補完しない）。

## 大会結果（/results）掲載ルール

結果ページは**網羅**が正解。予定（/events）の選別ルールを結果に適用しない。予定は選別・結果は網羅、の非対称は意図的。

### 方針

- **実際に開催され結果データがある大会は、主催・規模を問わず掲載する**。地方主催（THE BLACK BELT JAPAN / Torao Nation State / SECOND OUT 等）・DEEP地方IMPACT・DEEP JEWELS 等も含め、結果は残す。
- 理由：
  1. 結果ページはインデックス済みの資産。削除は404/リダイレクトを生みSEO資産を毀損する。
  2. 地方カード×選手名の長尾検索が本サイトの差別化（モート）。地方結果を削るとこれを自ら失う。
  3. 結果は fighterRecords（選手戦績）に紐づく。地方大会の結果を削ると戦績に穴が空き、戦績の正確性が崩れる。

### 例外（削除してよいのは品質理由のみ）

- 「開催されたが結果が空／薄い」ページ（thin content）は削除または補完の対象。
- これは"主要大会以外を抜く"ではなく"中身が無いページを抜く"。判定基準が別であることに注意。

### 結果まとめ「記事」との区別

- 上記は結果データの**掲載可否**の話。フル recap 記事（editorial まとめ）を書くかどうかは別基準（主要大会＋タイトルマッチ＋DB掲載選手が絡む試合）で選別してよい。
- 構造化された結果リスト＝網羅、手書きまとめ記事＝選別。混同しない。
