# VS対戦比較カード リデザイン仕様書

対象: `/events/*`（対戦カード比較ブロック）、`/dream`（夢のカード）、`/vs/{slugA}/{slugB}`（共有先ページ + OGP）
版: v1.0（2026-07-16）
前提: 既存の文字サイズ・情報構成は変更しない。見た目のみ刷新。

---

## 1. デザイン原則

- 左=赤コーナー、右=青コーナーの対称構造。UFC Tale of the Tape の文法に寄せる
- 比較は「1本の綱引きバー」で表現し、数字が絶対値・バーが相対優劣を担う
- サイト掲載はライト、SNS共有（OGP・共有カード）はダーク
- SatoriでのOGP生成と共用するため、**レイアウト計算はすべて決定的（JS実行に依存しない）**であること

## 3. カード構造（DOM順）

1. コーナーストリップ（高さ4px、左半分red / 右半分blue）
2. 階級ラベル（例: 女子スーパーアトム級（49.0kg）+ 右に伸びる罫線）
3. 選手名帯（grid: `1fr 40px 1fr`、中央にVSバッジ）
4. 比較スタット×2（戦績+勝率 / フィニッシュ率）各行 = 数値行 + 綱引きバー
5. 直近5戦（左右にW/Lドット5個ずつ）
6. 過去対戦（点線罫で区切り。0回なら非表示）
7. 共通対戦相手（既存仕様のまま。§8の名寄せ注意のみ）
8. フッター（左: Mニュースロゴ+MNEWS.JP、右: 共有ボタン）

## 4. 選手名帯 — 高さ崩れ対策【重要】

過去に長名で崩れた箇所。以下をすべて満たすこと。

### 4.1 レイアウト
- grid: `grid-template-columns: 1fr 40px 1fr; align-items: center;`
- VSバッジは固定 40×40px。名前セルの行数に関わらず常に垂直中央
- 名前セルは **固定 min-height = 2行分**（`min-height: calc(1.25em × 2)`）+ `display:flex; align-items:center`
  - 左セル `justify-content:flex-start`、右セル `flex-end`
  - 1行でも2行でもセルの高さが変わらないため、左右の行数差で帯全体が歪まない
- 最大2行。`-webkit-line-clamp: 2` + `overflow:hidden` をハードリミットとして併用

### 4.2 フォントサイズ段階（決定的関数）
文字数（サロゲート考慮の code point 数）で決める。CSSの自動縮小は使わない（Satori共用のため）。

```ts
function fighterNameSize(name: string): number {
  const len = [...name.replace(/\s/g, '')].length;
  if (len <= 7)  return 20; // 例: 朝倉未来、イ・イェジ
  if (len <= 11) return 17; // 例: ホベルト・サトシ・ソウザ(12字は下段)
  return 15;                // 例: ラジャブアリ・シェイドゥラエフ(14字)
}
```
- Web側は `data-len` 属性 or クラス（`.fname--m` `.fname--s`）で同じ閾値を適用
- font-weight 900、line-height 1.25 は全段階共通

### 4.3 改行制御
- 「・」の直後に `<wbr>` を挿入（`name.replaceAll('・', '・\u200B')` でも可）
- `overflow-wrap: anywhere` をフォールバックに指定
- 「直樹」「NOEL」など1〜4文字名は20pxのまま1行表示（追加処理不要）

### 4.4 検証必須ケース
| ケース | 選手例 |
|---|---|
| 最長カタカナ | ラジャブアリ・シェイドゥラエフ（14字） |
| 中黒なし長名 | 宇佐美正パトリック、ブラックパンサーベイノア |
| 左右で行数が違う | 直樹 vs アレクサンダー・ソルダトキン |
| 英字 | NOEL、SASUKE、YA-MAN、HIME |

## 5. 綱引きバー仕様

### 5.1 計算
```
shareA = A / (A + B)
```
- 表示幅: 左 = shareA、右 = 1 - shareA（間に3pxギャップ）
- **クランプ: shareA を 0.08〜0.92 に制限**（負け側の消失・角丸崩れ防止）
- 丸め: 小数1位まで

### 5.2 エッジケース
| 入力 | 表示 |
|---|---|
| A = B（80vs80含む） | 50:50。正しく「互角」を意味するのでそのまま |
| A = 0 かつ B = 0 | ゼロ除算。**バー全体を neutral 色 1本**にし、50%基準線も非表示 |
| 片方 0（100vs0等） | クランプにより 92:8 |
| 片方データ欠損 | バー非表示、数値は「—」 |

注: バーは**相対優劣のみ**を表す（5vs5 と 80vs80 は同じ見た目）。絶対値は数値行が担う。この役割分担を崩さないこと（例: 絶対値をバー長にマップする変更は不可）。

### 5.3 装飾
- バー高さ 8px、角丸: 外側 4px / 内側（境目側）1px
- 中央に 50% 基準線（1px、muted 45%不透明、上に `50%` ラベル 9px mono）
- 適用スタット: ①戦績行 = 勝率で分割、②フィニッシュ率行 = フィニッシュ率で分割

### 5.4 数値行
- 左: `16-7-0（勝率70%）` 赤 / 中央: ラベル（戦績・フィニッシュ率） / 右: `7-8-0（勝率47%）` 青・右揃え
- **左右とも読み順は「戦績→（勝率）」で統一**。右側で順序を反転しない
- 数字16px mono 800、括弧内11px

## 6. 直近5戦・過去対戦

- W/Lドット: 22px円、W=win色 / L=loss色 / 引分・NC=muted「D」「N」。左は左寄せ・右は右寄せ、下に「直近5戦」10pxラベル
- 過去対戦: `日付(mono 12px) + 勝者側Wドット(18px) + 勝者名(勝者コーナー色・700) + 大会名/決着(muted 12.5px)`。0回なら見出しごと非表示
- 折返し: `flex-wrap: wrap`（大会名が長い場合に2行目へ落とす。省略しない）

## 7. 共有・OGP

- 共有ボタン: フッター右。X intentは既存の `/vs/{slugA}/{slugB}` 共有導線を踏襲
- `/vs/{slugA}/{slugB}` のOGP画像: **本カードのダーク版レイアウトをSatoriで 1200×630 に生成**
  - フォントサイズ関数（§4.2）・バー計算（§5.1）を共用。倍率スケールのみ変える
  - 対角コーナーグロー（左上red 14% / 右上blue 14%のradial）を含める
  - キャッシュ: 選手どちらかの戦績更新でrevalidate
- og-default.png へのフォールバックは選手解決失敗時のみ

## 8. データ整合の前提（要修正）

- 同一選手の重複スラッグ（例: karshyga-dautbek / dautbek-karshyga）はカード生成前に名寄せ解決すること。放置すると共通対戦相手が「須田萌里」「須田萌里（2戦目）」のように別行分裂する
- 共通対戦相手で同一相手と複数回対戦している場合は**1行にまとめ、結果ドットを時系列で複数並べる**（W W）

## 9. 受け入れテスト

1. 80% vs 80% → バー50:50、数値は両方80%表示
2. 0% vs 0% → neutralバー1本、基準線なし
3. 100% vs 0% → 92:8 でクランプ
4. ラジャブアリ・シェイドゥラエフ vs 直樹 → 帯の高さ不変、VSバッジ垂直中央
5. モバイル幅375px / デスクトップ560pxの両方でバー・名前が崩れない
6. OGP画像とWebカードで分割比率・フォント段階が一致する
7. reduced-motion環境でアニメーション（入れる場合）が無効化される
